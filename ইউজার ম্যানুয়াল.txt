অবশ্যই! আপনার তৈরি করা "Forhad Notification System" প্রজেক্টটির জন্য একটি পূর্ণাঙ্গ ইউজার ম্যানুয়াল নিচে দেওয়া হলো। এই ম্যানুয়ালটি দুটি অংশে বিভক্ত: একটি অ্যাডমিনদের জন্য (যারা নোটিফিকেশন পাঠাবেন) এবং অন্যটি ডেভেলপারদের জন্য (যারা প্রজেক্টটি সেটআপ বা রক্ষণাবেক্ষণ করবেন)।

ইউজার ম্যানুয়াল: Forhad Notification System

সংস্করণ ১.০

ভূমিকা

Forhad Notification System হলো একটি ওয়েব-ভিত্তিক প্ল্যাটফর্ম যা আপনাকে সহজেই আপনার ওয়েবসাইটের ভিজিটরদের কাছে পুশ নোটিফিকেশন পাঠাতে সাহায্য করে। এই সিস্টেম ব্যবহার করে আপনি নতুন আপডেট, অফার বা গুরুত্বপূর্ণ বার্তা সরাসরি ব্যবহারকারীর কম্পিউটার বা মোবাইল ডিভাইসে পৌঁছে দিতে পারেন।

পর্ব ১: অ্যাডমিনদের জন্য (For Admins)

এই অংশটি তাদের জন্য যারা অ্যাডমিন ড্যাশবোর্ড ব্যবহার করে নোটিফিকেশন পাঠাবেন।

১.১. অ্যাডমিন ড্যাশবোর্ডে লগইন

আপনার সিস্টেম অ্যাডমিনিস্ট্রেটরের দেওয়া admin.html ফাইলটি আপনার কম্পিউটারের যেকোনো ওয়েব ব্রাউজারে (যেমন: ক্রোম, ফায়ারফক্স) খুলুন।

স্ক্রিনে একটি পাসওয়ার্ড বক্স আসবে। আপনার নির্ধারিত পাসওয়ার্ড (ডিফল্ট: admin) টাইপ করুন এবং "Login" বাটনে ক্লিক করুন।

১.২. ড্যাশবোর্ড পরিচিতি

লগইন করার পর আপনি মূল ড্যাশবোর্ডটি দেখতে পাবেন, যেখানে কয়েকটি প্রধান অংশ রয়েছে:

Total Subscribers: এখানে দেখা যাবে মোট কতজন ব্যবহারকারী আপনার নোটিফিকেশন পাওয়ার জন্য সাবস্ক্রাইব করেছে।

Clean Invalid Tokens: এই বাটনে ক্লিক করে আপনি সেই সব ব্যবহারকারীকে তালিকা থেকে মুছে ফেলতে পারবেন যারা নোটিফিকেশন ব্লক করে দিয়েছে বা আনসাবস্ক্রাইব করেছে। এটি আপনার সাবস্ক্রাইবার তালিকাকে পরিষ্কার রাখে।

Broadcast Message Tab: এই ট্যাবটি ব্যবহার করে সকল সাবস্ক্রাইবারকে একসাথে নোটিফিকেশন পাঠানো যায়।

Direct Message Tab: এই ট্যাবটি ব্যবহার করে তালিকা থেকে নির্দিষ্ট একজন সাবস্ক্রাইবারকে নোটিফিকেশন পাঠানো যায়।

১.৩. সকল ব্যবহারকারীকে নোটিফিকেশন পাঠানো (Broadcast Message)

"Broadcast Message" ট্যাবে ক্লিক করুন (এটি ডিফল্টভাবে খোলা থাকবে)।

নিচের ফর্মটি পূরণ করুন:

Title: নোটিফিকেশনের শিরোনাম দিন। (উদাহরণ: নতুন আপডেট!)

Body: নোটিফিকেশনের মূল বার্তাটি এখানে লিখুন। (উদাহরণ: আমাদের ওয়েবসাইটে নতুন ফিচার যোগ হয়েছে।)

Target URL: ব্যবহারকারী নোটিফিকেশনে ক্লিক করলে কোন ওয়েবসাইটে যাবে, সেই লিঙ্কটি এখানে দিন।

Icon URL (Optional): নোটিফিকেশনের বাম পাশে একটি ছোট ছবি (আইকন) দেখাতে চাইলে সেই ছবির URL এখানে দিন। এটি বর্গাকার (Square) হলে ভালো দেখায়।

Banner Image URL (Optional): নোটিফিকেশনের মূল লেখার নিচে একটি বড় ছবি (ব্যানার) দেখাতে চাইলে সেই ছবির URL এখানে দিন।

সব তথ্য ঠিকভাবে দেওয়ার পর "Send to All Subscribers" বাটনে ক্লিক করুন।

নিচে একটি সফল বার্তা (Successfully sent...) দেখতে পাবেন।

১.৪. নির্দিষ্ট ব্যবহারকারীকে নোটিফিকেশন পাঠানো (Direct Message)

নোটিফিকেশন ডিজাইন পরীক্ষা করার জন্য বা কোনো নির্দিষ্ট ব্যবহারকারীকে বার্তা পাঠানোর জন্য এটি ব্যবহার করুন।

"Direct Message" ট্যাবে ক্লিক করুন।

"Load" বাটনে ক্লিক করে সকল সাবস্ক্রাইবারের তালিকা লোড করুন।

ড্রপডাউন মেনু থেকে কাঙ্ক্ষিত ব্যবহারকারীকে (তার টোকেনের প্রথম অংশ দেখে) নির্বাচন করুন।

নিচের ফর্মটি (Title, Body, URL, Images) আপনার বার্তা অনুযায়ী পূরণ করুন।

সবশেষে "Send to Selected Subscriber" বাটনে ক্লিক করুন।

১.৫. অকার্যকর সাবস্ক্রাইবার পরিষ্কার করা (Cleanup)

ড্যাশবোর্ডের উপরের দিকে "Clean Invalid Tokens" বাটনে ক্লিক করুন।

একটি কনফার্মেশন বক্স আসবে। "OK" ক্লিক করে নিশ্চিত করুন।

সিস্টেম স্বয়ংক্রিয়ভাবে অকার্যকর টোকেনগুলো খুঁজে বের করে ডাটাবেস থেকে মুছে ফেলবে এবং নিচে একটি সফল বার্তা দেখাবে।

"Total Subscribers" সংখ্যাটি স্বয়ংক্রিয়ভাবে আপডেট হয়ে যাবে।

পর্ব ২: ডেভেলপারদের জন্য (For Developers/Setup Guide)

এই অংশটি তাদের জন্য যারা সিস্টেমটি প্রথমবার সেটআপ করবেন বা এর কোনো পরিবর্তন করতে চান।

২.১. প্রয়োজনীয় উপকরণ

Node.js: আপনার কম্পিউটারে Node.js ইনস্টল করা থাকতে হবে।

Firebase Account: একটি গুগল অ্যাকাউন্ট ব্যবহার করে Firebase-এ একটি প্রজেক্ট তৈরি করতে হবে।

GitHub Account: কোড অনলাইনে হোস্ট করার জন্য।

Render.com Account: সার্ভার বিনামূল্যে হোস্ট করার জন্য।

২.২. প্রজেক্ট সেটআপ (লোকাল কম্পিউটার)

প্রজেক্টের সম্পূর্ণ কোড একটি ফোল্ডারে (forhad-admin-project) রাখুন।

Firebase থেকে service-account-key.json ফাইল ডাউনলোড করুন:

Firebase Console-এ আপনার প্রজেক্টে যান।

Project Settings > Service accounts-এ যান।

"Generate new private key" তে ক্লিক করে service-account-key.json ফাইলটি ডাউনলোড করুন এবং প্রজেক্টের মূল ফোল্ডারে রাখুন।

২.৩. ফ্রন্টএন্ড ফাইল হোস্ট করা (GitHub Pages)

public ফোল্ডারের ভেতরের সমস্ত ফাইল (index.html, script.js ইত্যাদি) আপনার GitHub Pages রিপোজিটরিতে (your-username.github.io) আপলোড করুন।

২.৪. সার্ভার হোস্ট করা (Render.com)

প্রস্তুতি:

server.js ফাইলটি খুলুন এবং নিশ্চিত করুন এটি process.env.SERVICE_ACCOUNT_KEY এবং process.env.PORT ব্যবহার করছে।

.gitignore ফাইলে service-account-key.json এবং node_modules যোগ করা আছে কিনা তা নিশ্চিত করুন।

GitHub-এ আপলোড: সম্পূর্ণ প্রজেক্টটি একটি প্রাইভেট GitHub রিপোজিটরিতে আপলোড করুন।

Render-এ ডিপ্লয়:

Render.com-এ লগইন করে একটি নতুন "Web Service" তৈরি করুন এবং আপনার GitHub রিপোজিটরিটি কানেক্ট করুন।

Build Command: npm install

Start Command: npm start

Instance Type: Free

Environment Variable যোগ করুন:

Key: SERVICE_ACCOUNT_KEY

Value: আপনার service-account-key.json ফাইলের সম্পূর্ণ কনটেন্ট এখানে পেস্ট করুন।

"Create Web Service" এ ক্লিক করে ডিপ্লয়মেন্ট শুরু করুন।

২.৫. চূড়ান্ত কনফিগারেশন

Render-এ ডিপ্লয় সফল হলে, আপনি একটি লাইভ URL পাবেন (যেমন: https://your-app.onrender.com)।

আপনার লোকাল কম্পিউটারে থাকা admin.html ফাইলটি খুলুন।
